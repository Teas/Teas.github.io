发表于 2018-01-05

收费系统数据表简单操作

引用

该篇为简单数据表查询。暂时只涉及到应收账款 历史表 费项简单的查询。若有需求,可提出。
在操作数据库的时候,切记以下几点

  1. 在查询数据表的时候,任何情况下都需要加入查询条件。
    如SELECT * FROM tbc_receivable_record这种查询不可出现。
  2. 只可查询,若需删除数据联系相关开发人员。

相关表

  • 应收账款表 (tbc_receivable_record)
  • 费项(tbc_fee_item)
  • 历史表(tbc_receivable_history)
  • 流水表(tbc_money_flowrecord)

应收账款

tbc_receivable_record

常用属性
  • 开始(结束时间)时间
    start_time end_time
  • 关联费项
    fi_id
  • 关联历史表
    rh_id
    注:该属性关联最后一次操作的历史。若该账款结算两次,则关联最后次操作的历史
  • 状态
    state,settlement_status
    state:1-银行托收(冻结) 2-作废(销账) 3-挂起 （0或null为正常）; settlement_status=1为已结算
  • 关联房间
    building_id
通过receivableId查询应收账款

SELECT * FROM tbc_receivable_record where receivable_id=

通过费项Id和房间id查询

select * from tbc_receivable_record where fi_id= and building_id=

历史表

款项，押金，预收的收取记录
以账款为例,若同条账款结算两次,则会生成两条历史记录

tbc_receivable_history

  • 主键id
    rh_id
    该属性与应收账款的rhId对应,通过该属性能查询到关联的应收账款和流水表等信息
  • 单号
    receivable_num
    该属性与首页收款历史单号是同个属性
  • 是否已销账
    is_cancellation
    1为已销账
  • 收款时间
    receivable_time
已知rhid获取账款

select * from tbc_receivable_record where rh_id =

费项

tbc_fee_item

  • 主键id
    fi_id
  • 费项名称
    fi_name
    查询某个项目下的费项名称
    select fi_id,fi_name from tbc_fee_item where project_id=

流水表

每一笔应收账款每结算一次都会生成一笔流水记录。多次结算生成多笔流水记录。
预收、押金、冲抵、退款、销账、余额操作也会生成流水。

tbc_money_flowrecord

  • 主键id
    id
  • 关联历史表
    rh_id
  • 金额
    amount_paid
  • 关联楼栋
    building_id
  • 创建时间
    create_time
  • 关联应收账款
    receivable_id
    根据历史id获取流水
    select * from tbc_money_flowrecord where rh_id =

多表查询

多表查询可以简单理解为 从多张表里获取必要的信息
如:我已知收款历史id(rh_id),需要知道关联的账款信息以及费项名称、楼栋名称。
这个时候,就需要从账款表、费项表、楼栋表获取相关信息
